# 上线前两周执行清单（按天）

## 职责
- 提供试运行前 14 天可执行、可打勾、可追踪的落地清单。
- 将“联调、稳定性、发布、回滚、验收”串成统一节奏。

## 包含内容
- D-14 ~ D-1 的每日目标、执行项、产出物。
- Go/No-Go 上线判定与回滚触发条件。
- 角色分工建议（开发/测试/产品/运维）。

## 不包含内容
- 具体接口字段定义（见 `06`~`11`）。
- 服务器底层运维细节（见运维手册/环境文档）。

---

## 使用说明
- 建议每日站会按本清单逐项勾选。
- 每天结束需沉淀 3 个产物：`问题清单`、`结论记录`、`次日风险`。
- 阻塞项超过 24 小时未解除，必须升级处理（负责人 + 截止时间）。

---

## D-14 ~ D-12：环境与基线冻结

### D-14（环境一致性日）
- [ ] 确认 `staging` 配置与本地一致（DB/Redis/MinIO/JWT/API 前缀）。
- [ ] 校验 MinIO bucket、访问密钥、预签名有效期策略。
- [ ] 执行：`npm run test:e2e`（api）与 `npm run build`（web）。
- 产出：`环境核对表 v1`。

### D-13（数据与账号准备日）
- [ ] 准备联调学校测试数据（年级、班级、科目、教师、学生）。
- [ ] 创建联调账号并分发最小权限（管理员、教师）。
- [ ] 校验学校隔离、角色权限边界（特别是 audit 权限）。
- 产出：`联调数据集 v1`、`账号权限表`。

### D-12（流程冒烟日）
- [ ] 执行 `scripts/smoke-role-flow.sh`。
- [ ] 补充教师链路参数并验证 `403` 权限拒绝行为。
- [ ] 记录失败步骤、接口响应、复现方式。
- 产出：`角色冒烟报告 v1`。

---

## D-11 ~ D-8：主链路压测与缺陷收敛

### D-11（考试链路专项）
- [ ] 管理员完成：建档案 -> 建考 -> 科目配置。
- [ ] 校验异常路径：重复提交、非法参数、越权访问。
- [ ] 修复 P1/P2 缺陷并回归。
- 产出：`考试链路问题单`。

### D-10（阅卷链路专项）
- [ ] 教师完成：任务查看 -> 阅卷提交 -> 状态流转。
- [ ] 校验任务重复提交、防并发覆盖策略。
- [ ] 回归 marking 进度统计准确性。
- 产出：`阅卷链路问题单`。

### D-9（成绩与发布专项）
- [ ] 管理员完成：重算 -> 发布 -> 查询 -> 撤回。
- [ ] 校验总分、排名、发布状态切换一致性。
- [ ] 对关键统计口径进行人工复核抽样。
- 产出：`成绩口径校验记录`。

### D-8（文件与审计专项）
- [ ] 验证真实上传（`upload-binary`）与预签名下载。
- [ ] 验证文件删除、软删可见性、失败回退行为。
- [ ] 校验关键操作均落审计日志，字段完整。
- 产出：`文件审计闭环报告`。

---

## D-7 ~ D-4：体验优化与发布演练

### D-7（前端可用性修复）
- [ ] 统一 loading、空态、错误提示、成功反馈。
- [ ] 统一列表分页与筛选交互。
- [ ] 关键危险操作增加二次确认（删除/发布/撤回）。
- 产出：`前端体验优化清单`。

### D-6（性能与稳定性）
- [ ] 识别慢接口（查询、统计、列表）并优化。
- [ ] 检查索引命中、慢查询日志、热点接口吞吐。
- [ ] 形成性能基线（P95、错误率、超时率）。
- 产出：`性能基线报告`。

### D-5（发布演练 1）
- [ ] 在 staging 全流程演练：迁移 -> 启动 -> 冒烟 -> 回滚。
- [ ] 验证 CI/CD 产物可直接部署。
- [ ] 校验 secrets 未泄露、日志无敏感信息。
- 产出：`发布演练记录 v1`。

### D-4（发布演练 2）
- [ ] 按演练 1 的问题修复后重复演练。
- [ ] 验证数据库备份与恢复耗时满足目标。
- [ ] 更新操作手册与应急联系人。
- 产出：`发布演练记录 v2（通过）`。

---

## D-3 ~ D-1：上线决策准备

### D-3（UAT 收口）
- [ ] 业务方/UAT 按场景单回归并签字。
- [ ] 所有 P1 必须关闭，P2 需给出可接受风险说明。
- [ ] 冻结非必要需求，进入发布窗口。
- 产出：`UAT 验收单`。

### D-2（Go/No-Go 评审）
- [ ] 召开上线评审会（开发/测试/产品/运维）。
- [ ] 逐项过“上线判定标准”。
- [ ] 明确上线窗口、值班安排、回滚策略。
- 产出：`Go/No-Go 结论`。

### D-1（预发布锁定）
- [ ] 打版本标签并锁定发布分支。
- [ ] 再次执行完整冒烟 + 关键手工回归。
- [ ] 预置监控告警阈值与值班通知群。
- 产出：`发布前最终检查表`。

---

## D-Day：上线执行
- [ ] 按发布手册执行部署与迁移。
- [ ] 10~30 分钟内完成核心链路冒烟。
- [ ] 发布后首小时重点盯错误率、延迟、关键业务成功率。
- [ ] 异常触发回滚阈值即执行回滚，不做口头拖延。

---

## 上线判定标准（Go）
- [ ] P1 缺陷 = 0。
- [ ] 角色主链路（管理员/教师）均可闭环完成。
- [ ] `files + audit` 可追踪，关键操作可审计。
- [ ] 发布演练至少 2 次通过，且具备可验证回滚路径。
- [ ] 关键监控与告警已生效。

## 回滚触发条件（No-Go / Rollback）
- 任一核心链路不可用超过 10 分钟。
- 数据一致性出现不可接受偏差（总分、排名、发布状态）。
- 鉴权或数据隔离出现越权风险。
- 错误率或超时率连续超过阈值且 15 分钟内无法恢复。

---

## 推荐责任人分工
- 开发负责人：修复优先级、发布执行、技术决策。
- 测试负责人：用例覆盖、缺陷分级、回归结论。
- 产品/业务负责人：UAT 验收、上线窗口确认。
- 运维/平台负责人：环境稳定、监控告警、回滚执行。
