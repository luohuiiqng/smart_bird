# Go/No-Go 会议模板（上线决策）

## 职责
- 为上线前决策会议提供统一输入、评审流程与结论记录模板。
- 确保“可上线/延期/回滚预案”有据可依、责任明确。

## 包含内容
- 会议前输入材料清单
- 会议中评审问题清单
- 决策记录模板（Go / No-Go）
- 会后行动项模板

## 不包含内容
- 接口字段细节与代码实现说明
- 服务器底层操作命令

---

## 一、会前输入材料（必须齐全）
- [ ] 最新缺陷清单（P1/P2/P3）与状态
- [ ] 角色主链路冒烟结果（管理员/教师）
- [ ] 发布演练记录（至少 2 次）
- [ ] 性能基线与告警阈值
- [ ] 数据备份与回滚演练结果
- [ ] 本次发布范围（commit/tag/变更说明）

## 三、评审清单（逐项过）

### 1) 质量与功能
- [ ] P1 缺陷为 0
- [ ] P2 风险可接受并有补救计划
- [ ] 核心链路可用（登录、建考、阅卷、发布、审计、文件）
- [ ] 教师权限边界验证通过（审计接口 403）

### 2) 数据与安全
- [ ] 数据一致性抽样通过（总分/排名/状态）
- [ ] 学校隔离与越权测试通过
- [ ] 密钥/凭据未泄露到仓库与日志

### 3) 发布与回滚
- [ ] 发布手册可执行
- [ ] 回滚条件和触发阈值已确认
- [ ] 回滚责任人与操作顺序已明确
- [ ] 监控告警上线前已开启
