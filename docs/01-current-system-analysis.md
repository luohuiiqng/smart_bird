# 对标系统调研纪要（当前系统）

## 职责
- 仅记录对目标系统（`https://hubei.xfkj.cc/manage/index`）的客观调研结论。

## 包含内容
- 页面与模块观察结果
- 可见接口与链路线索
- 技术栈线索

## 不包含内容
- 自研系统需求定义
- 自研技术方案决策
- 数据库/接口最终设计

---

## 一、调研范围与方式
- 范围：登录页、首页、主功能入口页、考试与阅卷相关页、分析页。
- 方式：基于已登录会话进行页面抓取与接口观察（只读分析）。

## 二、产品定位结论
- 平台定位：学校场景的考试与阅卷一体化系统。
- 核心链路：建考 -> 组织阅卷 -> 成绩发布 -> 分析报表。
- 角色线索：管理员、教师、家长端入口（移动端提示可见）。

## 三、已识别功能模块

### 1) 主菜单级模块
- 应用中心
- 阅卷任务
- 本校考试管理
- 答题卡工具
- 用户指南
- 下载中心
- 系统日志

### 2) 应用中心快捷模块
- 本校考试
- 答题卡设计
- 成绩分析
- 学生档案
- 班级档案
- 多校联考
- 阅卷任务
- 教师管理
- 科目档案
- 年级档案
- 科目拆分

## 四、页面行为与流程线索
- 考试列表页可见多阶段状态：创建阶段、阅卷阶段、待发布阶段、已发布阶段。
- 阅卷任务页支持按阅卷状态筛选（全部、阅卷中、已完成）。
- 成绩分析页提供“原始分析”入口，存在跨系统授权跳转行为。

## 五、接口与技术线索

### 1) 登录与授权链路（观察结论）
- 登录请求触发后端接口：`/manage/GetData`（`cmd=login`）
- 成功返回：`url`、`appid`、`token`
- 再通过 `authpage.aspx` 完成认证跳转进入业务系统

### 2) 业务数据接口（观察结论）
- 页面公共请求入口：`/pages/Action.ashx`
- 通过 `code` 参数区分业务查询（如考试列表、阅卷任务列表）

### 3) 前后端技术线索
- 前端：jQuery + layui（页面脚本与资源可见）
- 后端：IIS + ASP.NET（响应头和页面痕迹）

## 六、样本数据观察（当前账号）
- 学校侧账号下可见考试数据规模较大（列表返回总量非零）。
- 科目维度存在并行状态推进，符合真实校内考试运行场景。

## 七、调研结论摘要
- 该系统是“考试组织 + 阅卷执行 + 成绩分析 + 基础档案”的完整闭环。
- 后续自研可优先围绕“学校管理员”主场景构建MVP，再扩展移动端与更复杂分析能力。
