# 项目文档总览

## 文档目标
- 统一沉淀“对标系统调研 + 自研系统设计”的关键信息。
- 保证文档职责单一，避免重复描述和多处维护。

## 文档规范
- 每份文档必须包含：`职责`、`包含内容`、`不包含内容`。
- 事实类信息与设计类信息分离：调研结论不与方案设计混写。
- 变更时优先更新对应职责文档，其他文档仅保留链接与引用。

## 文档清单

### 0) `00-development-standards.md`
- 职责：定义全栈开发与学校场景 UI 的生产级底线（禁止推测与写死业务逻辑等）。
- 不包含：各业务模块的接口字段级契约。

### 1) `01-current-system-analysis.md`
- 职责：记录对现有目标系统的客观调研结果（页面、模块、接口线索、技术栈线索）。
- 不包含：自研系统设计方案、表结构定稿、研发排期。

### 2) `02-product-scope-mvp.md`
- 职责：定义自研系统MVP范围、角色、功能边界与迭代目标。
- 不包含：技术实现细节、数据库字段级设计、接口字段定义。

### 3) `03-system-architecture.md`
- 职责：定义整体技术架构（NestJS / PostgreSQL / Redis / MinIO）与关键链路。
- 不包含：详细业务需求、完整接口清单、全部表字段明细。

### 4) `04-data-model-core.md`
- 职责：定义核心领域模型、主数据关系与状态枚举。
- 不包含：页面交互流程、部署细节、非核心扩展字段。

### 5) `05-api-blueprint-v1.md`
- 职责：定义V1阶段API分组与接口蓝图（按模块分层）。
- 不包含：数据库实现细节、前端页面原型、非V1接口。

### 6) `06-auth-module-spec.md`
- 职责：定义认证与账号模块（auth/users/schools）的接口契约与业务规则。
- 不包含：其他业务模块（考试、阅卷、分析）的详细契约。

### 7) `07-org-module-spec.md`
- 职责：定义基础档案模块（年级、班级、科目、教师、学生）的接口契约与数据规则。
- 不包含：认证机制细节、考试与阅卷流程契约。

### 8) `08-exam-module-spec.md`
- 职责：定义考试管理模块（考试计划、考试科目、发布状态流转）的接口契约与业务规则。
- 不包含：阅卷执行细节、成绩分析报表契约。

### 9) `09-marking-module-spec.md`
- 职责：定义阅卷任务模块（任务分配、阅卷提交、进度统计）的接口契约与业务规则。
- 不包含：考试创建细节、分析报表口径定义。

### 10) `10-scores-analysis-module-spec.md`
- 职责：定义成绩模块与分析模块（汇总、发布、查询、统计口径）的接口契约与业务规则。
- 不包含：阅卷提交细节、基础档案维护规则。

### 11) `11-files-audit-module-spec.md`
- 职责：定义文件模块与审计模块（上传下载、对象访问、操作留痕）的接口契约与业务规则。
- 不包含：考试阅卷业务流程、分析统计口径。

### 12) `12-delivery-plan-mvp.md`
- 职责：定义MVP交付排期、人天估算、阶段验收标准与风险应对。
- 不包含：接口字段设计与底层代码实现细节。

### 13) `13-ci-cd-spec.md`
- 职责：定义 CI/CD 流程（GitHub Actions）、分支策略、环境与密钥约定。
- 不包含：业务接口契约、服务器底层运维手册。
